## MARKING RUBRICS

## Criteria, Score and Descriptors

### Task 1: Database Integrity (20 Marks)

* **Excellent (9.0 – 10.0)**

  * All CREATE TABLE commands are accurate; all integrity constraints correctly applied. Screenshots complete, clear, and well-organized. Explanations are thorough and show strong understanding.
* **Good (7.0 – 8.5)**

  * Most tables and constraints are correctly implemented; minor errors. Screenshots provided and generally clear. Explanations show good understanding but lack depth.
* **Average (5.0 – 6.5)**

  * Some tables correctly created; constraints may be incomplete or partially wrong. Screenshots provided. Explanations are basic.
* **Need Improvement (3.0 – 4.5)**

  * Many constraints incorrect or missing. Screenshots unclear or incomplete. Explanations lack clarity and accuracy.
* **Poor (0 – 2.5)**

  * Incomplete or inaccurate SQL statements. Little or no screenshots. Weak or missing explanations.

**Weight/Marks:** 20

---

### Task 2: SQL Implementation (30 Marks)

* **Excellent (9.0 – 10.0)**

  * All remaining CREATE TABLE commands are fully correct and clearly shown with screenshots.
  * All tables populated with 6–10 valid rows each.
  * All SQL categories include 3 correct queries each, executed without errors.
  * Screenshots of queries and outputs are complete, clear, and well-organized.
  * Demonstrates excellent understanding of SQL, filtering, joins, arithmetic expressions, and formatting.
* **Good (7.0 – 8.5)**

  * CREATE TABLE commands mostly correct with minor errors.
  * Tables filled properly with 6–10 rows each.
  * Most SQL categories include 3 correct queries (some minor mistakes allowed).
  * Screenshots are provided but may lack full clarity or formatting.
  * Shows good understanding but misses depth in a few areas.
* **Average (5.0 – 6.5)**

  * Some CREATE TABLE commands incomplete or inaccurate.
  * Tables partially populated (some missing rows or incorrect data).
  * SQL categories include fewer than 3 correct queries or have notable syntax/output issues.
  * Screenshots provided but not consistent or clear.
  * Understanding is present but basic.
* **Need Improvement (3.0 – 4.5)**

  * CREATE TABLE commands contain multiple mistakes or missing screenshots.
  * Many tables not properly populated.
  * Several SQL categories missing or incorrectly executed.
  * Screenshots unclear, incomplete, or missing outputs.
  * Limited understanding of SQL syntax and application.
* **Poor (0 – 2.5)**

  * Missing or incorrect CREATE TABLE commands.
  * Very little or no data inserted.
  * SQL queries missing, incorrect, or not executed.
  * Screenshots mostly missing.
  * Shows minimal or no understanding of SQL concepts.

**Weight/Marks:** 30

### Task 3: Triggering (15 Marks)

* **Excellent (9.0 – 10.0)**

  * Both triggers fully correct and functional. BEFORE/AFTER results clearly demonstrated with proper screenshots. No syntax errors.
* **Good (7.0 – 8.5)**

  * Two triggers implemented with minor mistakes. Evidence provided for BEFORE/AFTER. Screenshots mostly clear.
* **Average (5.0 – 6.5)**

  * One trigger corrects, the second partially implemented. Limited or basic BEFORE/AFTER evidence.
* **Need Improvement (3.0 – 4.5)**

  * Triggers contain multiple errors; screenshots unclear or incomplete. Limited understanding shown.
* **Poor (0 – 2.5)**

  * Triggers missing, non-functional, or incorrect. No valid evidence provided.

**Weight/Marks:** 15

### Task 4: Access Control (15 Marks)

* **Excellent (9.0 – 10.0)**

  * Two access control mechanisms implemented correctly. Permissions tested and verified with clear screenshots. Demonstrates strong security understanding.
* **Good (7.0 – 8.5)**

  * Both mechanisms implemented with minor issues. Screenshots provided and mostly clear. Verification mostly correct
* **Average (5.0 – 6.5)**

  * One mechanism corrects, the other partially correct. Screenshots basic or incomplete.
* **Need Improvement (3.0 – 4.5)**

  * Incorrect or incomplete access control commands. Screenshots unclear or missing verification
* **Poor (0 – 2.5)**

  * Access control incorrectly implemented or not attempted. No valid evidence.
